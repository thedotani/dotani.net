# MOBILE-FIRST PREMIUM UI/UX ENGINE

## Purpose
Create ultra-premium, futuristic, mobile-first interfaces that are lightweight, responsive, and production-grade.

---

## Activation Triggers
- UI/UX tasks
- Styling or layout work
- Frontend design changes
- Animation or interaction design

---

## Core Philosophy

- Mobile-first is the default
- Desktop is an enhancement
- 99% users are mobile
- UX must be thumb-friendly and fast

---

## Layout System

- Start from 360px baseline
- Use vertical-first UX flow
- Use flex/grid + clamp + container queries
- Avoid fixed widths
- Minimal DOM depth

---

## UI Rules

- Clean spacing system (4px / 8px scale)
- Strong visual hierarchy
- One primary action per screen
- No clutter or template-like design

---

## HTML Rules

- Semantic HTML only
- Minimal wrapper elements
- Clean structural hierarchy

---

## CSS Rules

- Modern CSS only (grid, flexbox, clamp, variables)
- Avoid unnecessary utility spam
- Use CSS variables for theming
- Animations must be CSS-first

---

## JavaScript Rules

- Minimal JS only
- No JS unless necessary for real interaction
- Prefer native APIs

---

## Animation Rules

- Subtle and purposeful only
- Use transform + opacity only
- Avoid layout shifting animations
- Respect reduced-motion settings
- No scroll hijacking unless explicitly required

---

## Scroll Behavior

- Natural scrolling by default
- No forced smooth scroll unless requested
- Zero scroll jank tolerance

---

## Premium UI Standards

- Product-grade feel (not template design)
- Balanced whitespace usage
- Avoid overuse of shadows, blur, effects
- Typography must be clean and intentional

---

## Performance Constraints

- Minimal DOM
- Minimal JS
- Lazy load everything non-critical
- Avoid large animation libraries
- No unnecessary bundle growth
